배열
1. 정의와 성질
배열: 메모리 상에 원소를 연속하게 배치한 자료구조

배열의 성질
1. O(1)에 k번째 원소를 확인/변경 가능
2. 추가적으로 소모되는 메모리의 양(=overhead)가 거의 없음
3. Cache hit rate가 높음
4. 메모리 상에 연속한 구간을 잡아야 해서 할당에 제약이 걸림

확인/변경, 추가, 제거 = O(1)
임의의 위치에 원소를 추가, 제거 = O(N)
- 하나씩 밀어야 하기에, 원소를 연속해서 존재시키기 위해서

배열 사용 팁
전체를 특정 값으로 초기화할 효율적인 방법
1. cstring 헤더 memset함수(비추천)
- 실수할 여지가 굉장이 많음
-- 0, -1이 아닌 값을 넣은 경우
-- 2차원 이상의 배열을 함수의 인자로 넘긴 경우

2. for 문을 돌면서 값을 하나하나 다 바꾸는 방식
- 실수할 여지가 없기에 무난함

3. algorithm 헤더의fill 함수 이용(추천)
- fill 함수는 실수할 여지가 별로 없고 코드가 짧아 추천함

STL vector
- 배열과 거의 동일한 기능을 하는 자료구조
- vector는 배열과 달리 크기를 자유자재로 늘리거나 줄일 수 있는 장점
